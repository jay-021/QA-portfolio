Manual Testing Overview

A deep dive into the manual testing I conducted to ensure the quality and functionality of this portfolio website.

1. Functional Testing

I meticulously tested all website features:

  • -> Navigation menu links and their target pages.
  • -> Buttons like the "Resume Download" button.
  • -> Forms in the "Contact Me" section to ensure correct submission.
  • -> Interactive elements like image carousels and modals.

2. Usability Testing

I gathered feedback from friends and family on the website's ease of use:

  • -> I asked them to complete tasks like finding my contact information and navigating to specific project pages.
  • -> I observed their interactions and noted any points of confusion or difficulty.

3. Compatibility Testing

I verified the website's performance across different browsers and devices:

  • -> Tested on Chrome, Firefox, Safari, and Edge, ensuring layout and functionality remained consistent.
  • -> Checked on both Windows and macOS operating systems.
  • -> Used BrowserStack for broader browser and device compatibility testing.

4. Navigation Testing

I focused on the site's navigation flow:

  • -> Followed each link in the navigation menu to ensure it led to the correct page.
  • -> Checked for broken links or incorrect redirects.
  • -> Verified the mobile menu's functionality and responsiveness.

5. Regression Testing

After making updates, I retested critical functionalities:

  • -> Checked for new issues that may have been introduced by recent changes.
  • -> Re-validated form submissions and page loading times.
  • -> Reviewed any JavaScript interactions that might have been affected.

6. Cross-Device Testing

I tested the website's responsiveness on various devices:

  • -> Used my own collection of devices: desktop, laptop, tablet, and smartphone.
  • -> Simulated different screen sizes using browser developer tools.
  • -> Focused on elements like text resizing, image scaling, and touch interactions.

7. Content Testing

I carefully reviewed all website content:

  • -> Checked text for grammar, spelling, and clarity.
  • -> Verified the accuracy of information in the "About Me" section.
  • -> Ensured images and videos were loading correctly and displayed appropriately.

8. Performance Testing

I used online tools to assess website performance:

  • -> Google PageSpeed Insights and GTmetrix to analyze loading times and suggest optimizations.
  • Results of GTmetrix
  • -> Checked for large image files that might slow down the site.
  • -> Tested the website under simulated slow network conditions.

9. Security Testing

While not a security expert, I took basic precautions:

  • -> Ensured the contact form was protected against basic spam and bots.
  • -> Verified that sensitive information was not exposed in the website's code.
  • -> Considered using a web application firewall (WAF) in the future for enhanced security.

10. Localization Testing

Currently, my portfolio is in English, but I'm prepared for future localization:

  • -> I've researched localization best practices and tools.
  • -> If needed, I would translate content and adapt formats for specific regions.
