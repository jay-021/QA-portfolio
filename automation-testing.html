Automation Testing Overview

A deep dive into the automation testing I conducted to ensure the quality and functionality of this portfolio website.

1. End-to-End (E2E) Testing

I used Selenium WebDriver to simulate real user interactions and validate critical user flows, ensuring the website functions correctly from start to finish.

Selenium WebDriver Cucumber

2. UI Testing

Selenium helps me ensure the visual accuracy and responsiveness of my website's elements by testing their appearance and behavior.

Selenium WebDriver Cucumber

3. Component Testing

I utilize Selenium to isolate and test individual components like the contact form, ensuring they function correctly on their own.

Selenium WebDriver

4. Visual Regression Testing

I'm currently exploring open-source tools like Applitools Eyes and BackstopJS to automate the detection of visual changes on my website.

Applitools Eyes BackstopJS

5. Accessibility Testing

I use Selenium for automation and integrate tools like aXe or HTML CodeSniffer to verify that my website is accessible to everyone.

Selenium WebDriver aXe HTML CodeSniffer
